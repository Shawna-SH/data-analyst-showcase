You are a senior full-stack engineer. Build a Portfolio Showroom web app on Replit for a job seeker targeting Data Analyst / Data Engineer roles.

TECH STACK (preferred):
- Python 3.11
- Streamlit (multi-page)
- Plotly for charts
- pandas
- Optional: scikit-learn for a tiny demo model
No external paid APIs.

GOALS:
1) A clean, fast, professional portfolio “showroom” that can be shared in resume/interviews.
2) Support 3–5 featured projects now, and easy to add more later without touching Python code (data-driven content).
3) Include lightweight interactive demos:
   - Numeric prediction demo: a few text/number inputs -> predicted value
   - Image recognition demo: allow uploading ONE image only and optionally using the device camera (camera permission). Must limit file size and avoid allowing large uploads.
4) Include Resume/Links section: downloadable resume file link (placeholder), LinkedIn, GitHub, Email contact.
5) Make it stable on Replit.

INFORMATION ARCHITECTURE (PAGES):
- Home: short intro, roles targeted (Data Analyst / Data Engineer), quick links, featured projects (cards).
- Projects: searchable and filterable project list (tags), each project opens a detail page.
- Project Detail: structured case-study layout.
- Demos:
   - Numeric prediction (simple regression or rule-based baseline; keep fast)
   - Image recognition (use a lightweight pretrained model if feasible; otherwise stub with clear placeholder and architecture; include st.camera_input when available)
- About: skills, tools, and a short “what I build” summary.
- Resume & Contact: links + a “download resume” placeholder file path and contact buttons.

DATA-DRIVEN PROJECT CONTENT:
Store project metadata and content in a folder, e.g.:
- /content/projects/*.yml
Each YAML includes:
- id, title, short_summary, role (analyst/engineer), tags, tech_stack, dataset/source,
- problem, approach, key_results (bullets), visuals (optional image paths), links (github/demo/blog),
- “interview_talking_points” (bullets),
- “what_i_would_improve_next” (bullets)
The app should load all YAMLs at runtime and render them.

PROJECT TEMPLATES:
On project detail page, show sections in this order:
1) Overview (1 paragraph)
2) Problem & context
3) Data (source, size, key fields, quality checks)
4) Approach (methods/pipeline/model)
5) Results (metrics + key insights)
6) Visuals (Plotly charts and/or images if provided)
7) Engineering notes (logging, retries, dedup, schema, etc.) if relevant
8) Links (GitHub, demo, docs)
9) Interview talking points
10) Next improvements

UI REQUIREMENTS:
- Minimal, modern layout with consistent typography and spacing.
- Project cards with tags.
- Search box + tag filter.
- Include a “Featured” badge for selected projects.
- Dark mode is optional (only if easy).
- No heavy animations.

SECURITY / PERFORMANCE:
- Limit file upload size for image demo (e.g. max 2–5MB) and accept only jpg/png.
- Do not store uploaded files permanently; process in-memory only.
- Add caching where helpful.
- Ensure demos run quickly (avoid heavy model downloads).
- Include clear error handling and user-friendly messages.

DELIVERABLES:
1) Working Streamlit app with the above pages and navigation.
2) Sample project YAML files (3–5) with realistic content placeholders for:
   - EDA/analysis project
   - Web scraping + ETL/pipeline project
   - Dashboard project (even if not built yet, show how it will look)
   - ML numeric prediction project
3) A small sample dataset (or synthetic) for the numeric prediction demo.
4) Clear README with:
   - How to run on Replit
   - How to add a new project by adding a YAML file
   - Where to put images and resume PDF
5) Keep code clean, readable, and use clear variable names.

IMPORTANT:
- Do not ask me questions; make reasonable defaults.
- Use placeholder personal info fields at the top of a config file (name, headline, links) so I can edit easily.
- If camera input is not supported in the environment, gracefully fall back to upload only.
